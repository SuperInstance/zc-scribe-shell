## Step 2: Scoring Rubric (0-10)

*   **0-2 (Poor):** Missing or nearly empty README. No context, setup, or usage.
*   **3-4 (Basic):** Has a project name and brief description. May have broken or incomplete setup instructions.
*   **5-6 (Adequate):** Clear purpose, basic setup/installation steps, and a simple usage example. Structure is present but could be improved.
*   **7-8 (Good):** Well-structured with clear sections (Purpose, Installation, Usage, API, Contributing). Examples are practical and work. Links to related projects.
*   **9-10 (Excellent):** All "Good" criteria, plus: clear architectural diagrams, comprehensive API docs, troubleshooting guide, contribution guidelines with code standards, and a clear link to the larger fleet vision.

**Overall Fleet Documentation Health: Variable (Average Score: 5.2/10)**

The quality of documentation across the Cocapn fleet is inconsistent. There is a clear correlation between a project's centrality to the core vision and the quality of its docs.

*   **Core Projects (`plato-torch`)** are well-documented, providing enough context and examples for a developer to start contributing and understanding the fleet's primary metaphor.
