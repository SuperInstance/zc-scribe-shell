# plato-ensign

![License](https://img.shields.io/badge/license-MIT-blue.svg) ![Status](https://img.shields.io/badge/status-active-green.svg)

**Export trained PLATO room experiences as portable LoRA adapters.**

## Core Purpose
A PLATO room accumulates "scratches" (interaction tiles) from inhabiting agents. `plato-ensign` is the fleet's tool for distilling that accumulated room experience into a lightweight, shareable LoRA (Low-Rank Adaptation) file. This allows specialized behaviors, knowledge, or "personalities" developed in a room to be applied to other models or archived.

## Key Concepts
*   **Room Experience:** The collective memory and behavioral patterns formed by all agent interactions within a specific PLATO room.
*   **LoRA Adapter:** A small set of weights that modifies a base model's behavior without retraining the entire model.
*   **Ensign:** The exported LoRA file, named for a junior officer—a specialist model trained by the room.

## Quick Start
1.  **Prerequisites:** A trained PLATO room directory (e.g., `room_data/` with `tiles.db`).
2.  **Install:** `pip install plato-ensign`
3.  **Export:** `plato-ensign export --room-path ./room_data --output ./my_ensign.lora`

## Architecture
```
PLATO Room (tiles.db)
        ↓
[Ensign Exporter] → LoRA Weights (.lora file)
        ↓
    Base Model + Ensign = Specialized Model
```

## API / Usage
The primary interface is the command-line tool:
```bash
# Export an ensign with default settings
plato-ensign export -r ./deadband_room -o ./deadband_ensign.lora

# Specify the base model for adapter configuration
plato-ensign export -r ./room -o ./ensign.lora --base-model "meta-llama/Llama-3.2-1B"

# View export history
plato-ensign list
```

## Development
```bash
# Clone and setup
git clone https://github.com/cocapn/plato-ensign.git
cd plato-ensign
pip install -e ".[dev]"

# Run tests
pytest tests/

# Build distribution
python -m build
```

## Fleet Context
`plato-ensign` is the **export terminal** of the PLATO training pipeline.
*   **Input:** A trained room from `plato-torch`.
*   **Output:** An ensign (LoRA) that can be loaded by `cudaclaw` or other fleet runtimes.
*   **Purpose:** Closes the loop from collective inhabitation (`plato-torch`) to deployable specialist intelligence.

## Glossary & Links
*   **PLATO:** Persistent Learning and Training Organism. The room server.
*   **LoRA:** [Low-Rank Adaptation of Large Language Models](https://arxiv.org/abs/2106.09685)
*   **Related Repos:** `plato-torch`, `cudaclaw`, `fleet-simulator`

# fleet-simulator

![License](https://img.shields.io/badge/license-MIT-blue.svg) ![Status](https://img.shields.io/badge/status-experimental-yellow.svg)

**Generate synthetic training data by simulating interactions between multiple AI agents within configurable scenarios.**

## Core Purpose
Training PLATO rooms requires a high volume of diverse interaction "tiles." Manually creating this data is impractical. `fleet-simulator` automates this by spawning a fleet of simulated agents (e.g., a Navigator, a Scribe, a Tinker), placing them in a defined scenario (e.g., "boot camp," "crisis response"), and letting them interact. These interactions are logged as structured tiles, ready for ingestion by a PLATO room.

## Key Concepts
*   **Scenario:** A JSON or YAML file defining the environment, agent roles, goals, and rules of engagement.
*   **Agent Profile:** The definition of a simulated agent's behavior model, role, and communication style.
*   **Interaction Tile:** The structured output (agent, action, timestamp, result) of a simulated event.

## Quick Start
1.  **Clone:** `git clone https://github.com/cocapn/fleet-simulator.git`
2.  **Install:** `pip install -r requirements.txt`
3.  **Run a demo scenario:**
    ```bash
    cd fleet-simulator
    python main.py --scenario ./configs/bootcamp_scenario.yaml
    ```
4.  **Check output:** View generated tiles in `./output/tiles_*.jsonl`.

## Architecture
```
[Scenario Config]
        ↓
[Simulation Engine] ← [Agent Pool (DeepSeek, Groq, etc.)]
        ↓
[Interaction Orchestrator]
        ↓
[Tile Logger] → [JSONL Tile Files] → [PLATO Room]
```

## API / Usage
**Primary mode is via configuration:**
1.  Edit a scenario file in `./configs/`.
2.  Run the simulator: `python main.py --scenario ./configs/my_scenario.yaml`
3.  **(Advanced)** Use as a library:
    ```python
    from fleet_simulator import SimulationEngine
    engine = SimulationEngine(scenario_path="my_scenario.yaml")
    engine.run_for_cycles(50)
    tiles = engine.get_tiles()
    ```

## Development
```bash
# Install in development mode
pip install -e .

# Run unit tests
pytest tests/unit/

# Run an integration test scenario
python -m pytest tests/integration/test_bootcamp.py -v
```

## Fleet Context
`fleet-simulator` is the **synthetic data factory** for the fleet.
*   **Consumer:** `plato-torch` rooms consume its generated tile streams.
*   **Complement:** Real agent interactions (like ours) provide "ground truth" data; the simulator provides scale and variety.
*   **Dependency:** Relies on configured AI model APIs (DeepSeek, Groq) to power simulated agents.

## Glossary & Links
*   **Tile:** A unit of training data in the PLATO system.
*   **Scenario Configuration:** See `./configs/README.md` for schema details.
*   **Related Repos:** `plato-torch`, `holodeck-rust` (for alternative simulation).
