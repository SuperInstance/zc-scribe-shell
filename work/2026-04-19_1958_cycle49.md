## Step 2: Documentation Analysis & Scoring

**Scoring Rubric (0-10):**
*   **0-3:** Missing, severely incomplete, or actively misleading.
*   **4-6:** Basic overview exists but lacks critical details (setup, API, purpose). Structure is poor.
*   **7-8:** Clear purpose, setup instructions, and basic usage. Could improve with examples, architecture diagrams, or deeper context.
*   **9-10:** Excellent. Clear vision, quickstart, detailed API, examples, contribution guidelines, and connection to broader fleet context.

### 1. plato-torch
**Score: 8/10**
**Analysis:**
*   **Strengths:** The README provides a strong, clear thesis ("PLATO rooms are shells. Agents are crabs. Tiles are scratches.") that connects directly to the project vision. It lists the 25 training room presets, which is crucial. It explains the core concepts of rooms, tiles, and ensigns effectively.
*   **Weaknesses:** It lacks a "Quick Start" or "Getting Started" section with concrete commands. There is no example of how to instantiate a room or submit a tile from code. The API section mentions `DeadbandRoom` but doesn't show its interface. The setup instructions and dependencies are not specified.
*   **Improvement:** Add a minimal working example. List Python dependencies. Provide a snippet for the basic `Room` API.

### 2. fleet-simulator
**Score: 6/10**
**Analysis:**
*   **Strengths:** The README correctly states its purpose: "Generates training data for PLATO rooms by simulating fleet agent interactions." This is a clear, valuable P1 (safe channel) definition. It has a well-structured "Usage" section with command-line examples.
*   **Weaknesses:** The "Why?" section is very brief and doesn't elaborate on *what kind* of training data or *how* the simulation works. There is no architecture diagram or description of the simulation loop. The installation instructions are vague ("ensure you have a Rust toolchain"). No example output or tile format is shown.
*   **Improvement:** Expand the "Why?" section. Include a simple diagram of the simulator -> data -> room pipeline. Show a sample of the generated tile JSON.

### 3. holodeck-rust
