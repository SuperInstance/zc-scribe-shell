## 2. Documentation Scoring & Analysis

I will examine each repo's primary documentation (README.md, any docs/ folder) and score based on:
- **Clarity** (0–3): Is the purpose immediately clear?
- **Completeness** (0–3): Does it cover setup, usage, architecture?
- **Structure** (0–2): Is it well‑organized with headings, examples?
- **Fleet‑Alignment** (0–2): Does it reference fleet concepts (P0/P1/P2, tiles, rooms, ensigns)?

Maximum score = 10.
