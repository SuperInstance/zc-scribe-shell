# Fleet Simulator
**Purpose:** Generate training data through multi-agent fleet simulations.

## Overview
The Fleet Simulator creates synthetic interaction data for training PLATO rooms and ensigns. It models agent behaviors, communication patterns, and task execution in a controlled environment.

## Key Features
- Multi-agent simulation with configurable behaviors
- Generates interaction "tiles" for room training
- Export to PLATO room compatible formats
- Configurable scenario templates

## Getting Started

### Prerequisites
- Rust toolchain (latest stable)
- Git

### Installation
```bash
git clone <repo-url>
cd fleet-simulator
cargo build --release
```

### Basic Usage
```bash
# Run a default simulation
cargo run -- --scenario default

# Generate 1000 interaction tiles
cargo run -- --tiles 1000 --output ./tiles/

# View available scenarios
cargo run -- --list-scenarios
```

## Configuration
Edit `config/simulator.toml` to adjust:
- Agent count and types
- Communication protocols
- Task complexity
- Output formats

## Architecture
- **Simulation Engine:** Discrete event scheduler
- **Agent Models:** Behavior trees with Deadband Protocol adherence
- **Tile Generator:** Formats interactions for PLATO consumption
- **Exporter:** Converts to room-specific formats

## Integration
- Feeds `plato-torch` training rooms
- Compatible with `holodeck-rust` NPC sentiment data
- Exports to `fleet-knowledge` for analysis

## Development
```bash
# Run tests
cargo test

# Build documentation
cargo doc --open

# Check for errors
cargo check
```

## Contributing
1. Fork the repository
2. Create a feature branch
3. Add tests for new scenarios
4. Submit pull request

## License
MIT
```
