## Step 1: Repository Selection & Scoring

I have selected 5 repositories from the fleet. I will read their primary documentation (README.md, any other top-level docs) and score them based on a standard rubric.

**Scoring Rubric (0-10):**
*   **0-2:** Missing or completely non-functional.
*   **3-4:** Exists but is minimal, outdated, or confusing.
*   **5-6:** Covers basics (what it is, how to run). Lacks context, examples, or clear structure.
*   **7-8:** Clear, structured, explains purpose and basic usage. May lack advanced details or integration context.
*   **9-10:** Excellent. Clear purpose, setup, usage, examples, architecture, and connection to the wider fleet.

## Step 2: Summary Report

**Overall Fleet Documentation Health: Below Average.**

*   **Average Score:** 5.2/10
*   **Best in Class:** `plato-torch` (8/10). Serves as a good model for other repos.
*   **Biggest Gap:** Repositories that are frameworks or libraries (`constraint-theory-core`, `cudaclaw`) have almost no usable documentation. This creates a high barrier to entry for contributors and hinders internal reuse.
*   **Common Pattern:** Most READMEs answer **"What?"** but poorly address **"Why?"** (fleet context) and **"How?"** (concrete examples, integration).

**Root Cause:** Documentation is treated as a final, static artifact rather than a "scratch" accumulated through use (aligning with the hermit crab metaphor). There's no evident process for updating docs as code evolves.

**Recommendations (P1 - Find Safe Channels):**
1.  **Template:** Create a fleet-standard `README.md` template in `fleet-knowledge/` with required sections: Purpose, Fleet Context, Quick Start, Detailed Usage, API/Examples, and Integration.
2.  **Ownership:** Assign each repo a "Doc Steward" role (could be a Scribe agent cycle) responsible for a quarterly doc review.
3.  **Integration Notes:** Mandate a "Fleet Context" section in every README, explaining how the project connects to at least one other fleet component (e.g., "This MUD server produces JSON logs consumed by `plato-torch` Room X").
4.  **Fix Broken Links:** A simple, fleet-wide scan for broken `*.md` links is a high-impact, low-effort P0 task.

**Proposed Next Task for Scribe:** Use `plato-torch` as a model to rewrite the README for `holodeck-rust`, adding a fleet context section and a concrete usage example.
